# Footbag Results Pipeline ‚Äî v1.0.17

Archive-quality, deterministic pipeline for building a canonical Footbag historical
results dataset and Excel workbook from an offline HTML mirror.

---

## Quick Start

```bash
git clone https://github.com/JamesLeberknight/footbag-results.git
cd footbag-results

make setup                  # create venv + install deps + mkdir out/

# Download mirror.tar.gz from the Releases page, then:
tar -xzf mirror.tar.gz      # extracts ‚Üí mirror/

make rebuild                # parse mirror ‚Üí out/stage2_canonical_events.csv
make release                # identity-locked outputs + workbook
make qc                     # data-integrity + schema/logic checks
```

All pipeline stages have documented entry points. `make all` runs the full chain.

---

## Design Goals

- Deterministic builds ‚Äî same inputs always produce the same outputs
- No silent merges ‚Äî every identity decision is human-verified
- Explicit identity overrides ‚Äî canonical truth lives in `inputs/identity_lock/`
- Reproducible from a clean clone + released mirror archive
- Clear separation: code / configuration / generated outputs / large static inputs

## Build Modes

The pipeline has two distinct modes. Do not mix them.

### Rebuild Mode (parse mirror ‚Üí canonical events)

Parses the HTML mirror and reconstructs events. Produces `out/stage2_canonical_events.csv`,
which is required input for Release Mode.

```bash
make rebuild
# or step by step:
python scripts/rebuild/01_parse_mirror.py
python scripts/rebuild/01b_import_old_results.py
python scripts/rebuild/01c_merge_stage1.py
python scripts/rebuild/02_canonicalize_results.py
```

### Release Mode (identity-locked canonical outputs)

Consumes identity-lock artifacts. Produces the deterministic canonical dataset and workbook.
Requires `out/stage2_canonical_events.csv` to exist (run Rebuild Mode first).

```bash
make release
# or step by step:
python 02p5_player_token_cleanup.py \
  --identity_lock_placements_csv inputs/identity_lock/Placements_ByPerson_v33.csv \
  --out_dir out
python 03_build_excel.py
python 04_build_analytics.py
```

Stage 04 writes `out/persons_truth.lock` ‚Äî the identity immutability sentinel.

> **Stage ordering:** Never re-run Stage 03 after Stage 04 ‚Äî it resets workbook sheet ordering.

## Identity Model (v1.0.17)

Identity is locked in `inputs/identity_lock/` and never recomputed during Release Mode.

- `Persons_Truth_Final_v31.csv` ‚Äî 3,451 canonical persons, collision-free
- `Persons_Unresolved_Organized_v27.csv` ‚Äî 76 genuinely ambiguous entries
- `Placements_ByPerson_v33.csv` ‚Äî 25,679 authoritative placements

Any identity change requires a new major version (v2.0.0) per `CLAUDE.md`.
