🚀 Build Modes

The pipeline has two distinct modes. Do not mix them.

## Release Mode (Canonical / Archive-Safe)

Consumes identity-lock artifacts. Produces the deterministic canonical dataset and workbook.

**Prerequisite:** `out/stage2_canonical_events.csv` must exist. Generate it by running
Rebuild Mode first (see below), or restore it from a prior run before proceeding.

```
python 02p5_player_token_cleanup.py \
  --identity_lock_placements_csv inputs/identity_lock/Placements_ByPerson_v25.csv \
  --out_dir out

python 03_build_excel.py       # requires out/stage2_canonical_events.csv

python 04_build_analytics.py   # writes out/persons_truth.lock (identity sentinel)
```

See `RELEASE_CHECKLIST.md` for the full release procedure.

## Rebuild Mode (Exploratory / Mirror Parsing)

Parses the HTML mirror and reconstructs events. Does NOT produce canonical identity.
Output (`out/stage2_canonical_events.csv`) is required as input to Stage 03.

```
python scripts/rebuild/01_parse_mirror.py
python scripts/rebuild/01b_import_old_results.py
python scripts/rebuild/01c_merge_stage1.py
python scripts/rebuild/02_canonicalize_results.py
```

⚠️ Stage Ordering Notes

- Stage 03 builds the base Excel workbook and requires `out/stage2_canonical_events.csv`
  (generated by Rebuild Mode stage 02).
- Stage 04 must run last. It finalizes analytics, enforces identity immutability,
  and writes the `persons_truth.lock` sentinel.
- **Never re-run Stage 03 after Stage 04** — this resets workbook sheet ordering.

🔐 Identity Model (v1.0.4)
Persons_Truth is guaranteed to be:

Collision-free

Deterministic

Derived from overrides + validated data

Never manually edited

If multiple effective_person_id values map to the same person_canon,
those rows are automatically moved to:

Persons_Unresolved

No speculative merges are performed.

The scripts below (`06`, `07`) are preserved in `legacy/` for reference.
